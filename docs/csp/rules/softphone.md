
## 測試情境：同一分機由不同帳號登入

1. 先以 A 帳號開啟軟體電話。
2. 再以 B 帳號開啟軟體電話。
3. B 帳號按下確認後，畫面會切換到 B 帳號。
4. 若 A 帳號當時正在通話，通話仍可正常進行，但軟體電話會跳到 B 帳號畫面；此時仍會產生小結。
5. B 帳號再次點擊軟體電話按鈕後，即可開啟並使用軟體電話。

## 常見 CTI 錯誤與處理

| 錯誤 | 原始訊息/原因 | 建議處理 |
|---|---|---|
| CTI-470 | `TERR_PARTY_NOT_ON_CALL`；請求中的對象不在通話中 | 不定時出現且通常不影響操作；若持續出現，向中和 3535 報備 |
| CTI-527 | 帳號已在其他分機登入 | 確認原電腦是否已登出 |
| CTI-56 | `TERR_INV_CONNID`；分機連線 ID 無效 | 確認該分機是否已有其他帳號登入 |
| CTI-565 | `TERR_INVALID_STATE`；狀態無效 | 依當下狀態重新登入或回報 |
| CTI-570 | 系統暫時卡住 | 登出後重新登入 |
| CTI-58 | `TERR_OUT_OF_SERVICE`；話機或 T-Server 異常 | 重新插拔實體話機連接線，等待約 1 分鐘重啟 |
| CTI-607 | 原文件未記載明確原因 | 蒐集時間、帳號及 log 後回報 |
| CTI-61 | `TERR_INV_CALL_DN`；撥號 DN 無效 | 稍後重新登入 |
| CTI-621 | 專員撥打內線 | 確認撥打內線原因；通常不影響操作 |

其他操作注意事項：誤開測試區時關閉後重新開啟正式區；個人系統同時最多保留 4 個網頁。軟體電話異常 log 可由 `/Api/WriteLog` 查詢，原始文件中的 Kibana 連結未在此重複列出。
